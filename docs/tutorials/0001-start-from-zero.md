# 0001：纯小白从 0 开始学 AIOps

今天只学一件事：AIOps 到底在做什么，并写出第一张 AIOps 案例卡。

## 1. 先记住一句话

AIOps 不是“让 AI 自动修好一切”。

AIOps 是：用运维数据帮助人更快发现问题、理解问题、选择动作，并在安全边界内自动化。

## 2. AIOps 的四层地图

| 层级 | 你要理解什么 | 例子 |
|---|---|---|
| 系统 | 问题发生在哪里 | 服务器、应用、数据库、网络、Kubernetes |
| 数据 | 系统留下了什么证据 | 指标、日志、链路追踪、告警、变更记录 |
| 分析 | 如何从证据里看出异常 | 异常检测、告警聚合、事件关联、根因线索 |
| 动作 | 如何安全处理问题 | 通知、建议、runbook、人工确认、自动执行、回滚 |

纯小白不要一上来学算法。先学会看懂系统和数据。

## 3. 今天的 45 分钟任务

### 第 1 步：认识 6 个词

- 指标：数字，例如 CPU、内存、错误率、延迟、QPS。
- 日志：系统发生过什么的记录。
- 链路追踪：一次请求经过了哪些服务、哪里慢。
- 告警：系统提醒你某个条件异常。
- 事件：需要人处理的一组问题。
- Runbook：处理问题的步骤手册。

### 第 2 步：选一个故障

选一个你见过或能想象的故障：

- 网站打不开。
- 接口变慢。
- 磁盘满了。
- 数据库连接数打满。
- 发布后大量报错。

小故障也可以。越具体越好。

### 第 3 步：写案例卡

新建文件：

```text
docs/learning-records/2026-07-01-first-aiops-case.md
```

复制这个模板：

```md
# 第一个 AIOps 案例卡

## 故障现象
用户看到什么？业务受到什么影响？

## 当时能看到的数据
有哪些指标、日志、告警、变更记录？

## 人是怎么判断的
你或同事当时怎么一步步定位？

## 如果做成 AIOps
系统应该自动发现什么？合并什么？提示什么？

## 可以安全自动化的动作
先写低风险动作：通知、生成摘要、推荐 runbook。不要一上来自动重启生产服务。

## 我今天还不懂的地方
列 1-3 个问题，下一次问 agent 或查资料。
```

## 4. 今天的过关标准

你完成下面 3 件事就算过关：

- 能说清：AIOps 是数据、分析和安全自动化，不是魔法 AI。
- 能写出：一张故障案例卡。
- 能提交：把案例卡放进 GitHub 仓库。

## 5. 明天学什么

明天学“指标是什么”：CPU、内存、错误率、延迟、QPS 为什么能帮助发现问题。我们先用表格模拟，不急着装 Prometheus。
