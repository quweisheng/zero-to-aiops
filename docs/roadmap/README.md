# AIOps 学习路线

这条路线按“转岗作品集”设计，不按教材目录设计。每一阶段都要产出可以放到 GitHub 的证据。

## 0. 起点盘点

先写清楚自己的已有能力：

- Linux：排障、性能、网络、系统服务。
- 脚本：Shell、Python、自动化任务。
- 监控：是否用过 Zabbix、Prometheus、Grafana、ELK、SkyWalking、APM。
- 云原生：Docker、Kubernetes、CI/CD、发布。
- 故障经验：至少 3 个真实故障故事。

## 1. 第 1-2 周：SRE 思维

目标：把“会运维”转成“会定义可靠性目标并减少重复劳动”。

- 学 SLI、SLO、错误预算。
- 学 toil，找出自己工作里最值得自动化的 3 类重复劳动。
- 学事件复盘，写无责复盘模板。
- 产出：3 篇故障案例卡，1 篇 toil 清单。

验收标准：

- 能解释一个服务的核心 SLI。
- 能说清楚一个告警为什么有业务意义。
- 能把一个手工处理动作拆成输入、判断、动作、回滚。

## 2. 第 3-5 周：可观测性基础

目标：让指标、日志、链路和告警形成闭环。

- Prometheus：采集、PromQL、Alertmanager。
- Grafana：仪表盘、变量、告警视图。
- OpenTelemetry：Collector、trace、metric、log 的关系。
- 日志：结构化日志、日志模板、错误聚合。
- 产出：可观测性实验室。

验收标准：

- 一个 demo 服务有 metrics、logs、traces。
- 一个仪表盘能看出延迟、错误率、吞吐量和资源使用。
- 一个告警能触发，并且能写出处理 runbook。

## 3. 第 6-8 周：Python 数据分析与异常检测

目标：能用运维数据做第一个可解释模型。

- Python：pandas、numpy、matplotlib。
- 特征：滑动窗口、均值、方差、P95、错误率、日志模板计数。
- 基线：固定阈值、动态阈值、同比/环比。
- 模型：IsolationForest 或简单时序异常检测。
- 产出：指标异常检测器。

验收标准：

- 能读取样例指标数据并输出异常时间段。
- 能解释误报和漏报。
- 能把检测结果转成告警建议，而不是只画图。

## 4. 第 9-12 周：AIOps 项目化

目标：完成能写进简历的项目。

- 告警降噪：相同服务、相同依赖、相同时间窗口聚合。
- 根因线索：变更记录、拓扑依赖、错误日志、资源指标关联。
- Runbook：推荐动作、人工确认、执行记录、回滚。
- 产出：告警降噪器或 Runbook 助手。

验收标准：

- 项目 README 能让别人 10 分钟内跑起来。
- 有架构图、输入输出样例、效果对比和局限性。
- 面试时能讲清“为什么这样设计，风险怎么控制”。

## 5. 第 13-24 周：求职强化

目标：把学习成果转成岗位匹配度。

- 每周投递 5-10 个岗位，并记录关键词。
- 每周做 2 次面试题复盘。
- 每两周改一次简历项目表达。
- 补齐 Kubernetes、CI/CD、云平台、数据库、中间件等岗位共性要求。

验收标准：

- 简历有 2 个 AIOps 相关项目。
- 能讲 8 个运维工程故事。
- GitHub 最近 3 个月有持续提交记录。
