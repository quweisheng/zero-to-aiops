# AIOps 知识库精讲升级设计

## 背景

这个仓库的目标不是只保存链接清单，而是记录一个运维工程师从 0 转向 AIOps 工程师的完整学习路径。参考优秀技术知识库的做法，首页要像地图，单篇文章要像教程，学习记录要能证明自己真的做过实验。

## 目标

本次先完成一个小批次升级：

- 建立统一的技术栈精讲写作标准。
- 修复当前乱码的核心文章。
- 把 Prometheus、Grafana、Docker Compose 写成示范级文章。
- 在首页和技术栈总览里暴露“精讲标准”和第一批示范文章入口。

## 内容原则

- 只基于官方文档和真实运维学习场景做原创中文讲解。
- 不复制官方全文，不把官网内容机械翻译成中文。
- 每篇文章都要回答：为什么学、是什么、原理、架构、如何配置、如何验证、常见坑、AIOps 中怎么用。
- 每篇文章都要给零基础读者一个可执行的入门实验。
- 每篇文章最后都要给“学习证据”，方便后续记录到 GitHub。

## 信息架构

技术栈文章统一采用以下结构：

1. 学习目标
2. 官方资料
3. 为什么要学
4. 是什么
5. 核心原理
6. 架构和数据流
7. 安装与启动
8. 配置详解
9. 常用命令或查询
10. AIOps 场景
11. 入门实验
12. 常见故障排查
13. 学习检查清单
14. 面试题
15. 学习证据

## 第一批示范文章

- `docs/tech-stack/observability/prometheus.md`
- `docs/tech-stack/observability/grafana.md`
- `docs/tech-stack/cloud-native/docker-compose.md`

这三篇覆盖一个最小可观测性实验闭环：用 Docker Compose 启动环境，用 Prometheus 采集指标，用 Grafana 展示指标。

## 验证方式

- 运行 `npm run docs:build`，确保 VitePress 能正常构建。
- 使用 `rg` 检查乱码特征字符，确保修复后的示范文章不再包含明显 mojibake。
- 使用 `git diff --check` 检查 Markdown 中的空白问题。
