# AIOps 面试准备

## 自我介绍结构

1. 我有多年运维经验，熟悉生产系统稳定性、监控告警、故障处理和自动化。
2. 我现在把经验升级到 AIOps：用可观测数据、规则/模型和自动化流程降低 MTTD、MTTR 和告警噪声。
3. 我做了几个公开项目：可观测性实验室、指标异常检测器、告警降噪器。
4. 我的优势是知道生产动作的风险边界，自动化会设计审计、人工确认和回滚。

## 必备问题

- AIOps 和传统监控有什么区别？
- 指标、日志、链路追踪分别解决什么问题？
- Prometheus 的 pull 模型有什么优缺点？
- 什么是 SLI/SLO？如何设计一个服务的 SLO？
- 为什么告警不能只按 CPU 阈值配置？
- 如何降低告警误报？
- 如何判断一个异常检测模型是否有价值？
- IsolationForest 适合什么场景？不适合什么场景？
- 你如何把 runbook 自动化？风险如何控制？
- 出现模型误判导致错误动作怎么办？
- Kubernetes 里的 Deployment、Service、Ingress、HPA 分别做什么？
- 一次生产故障你会如何复盘？

## 8 个工程故事

- 最复杂的一次故障定位。
- 最有价值的一次自动化。
- 一次监控告警改造。
- 一次性能或容量问题。
- 一次发布或变更事故。
- 一次跨团队协作。
- 一次误报/漏报治理。
- 一个你做过或准备做的 AIOps 项目。

## 回答原则

- 先讲场景，再讲动作，最后讲结果。
- 多说数据：告警数量、处理时间、误报率、故障影响、自动化节省时间。
- 不把 AI 说成万能；要讲边界、评估、审计、回滚。
- 年龄不是重点，生产经验和风险意识才是重点。
